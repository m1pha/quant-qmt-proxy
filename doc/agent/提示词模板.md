### REST API 端点

#### 行情接口
```
GET /api/v1/quote/{symbol}

#### 交易接口
```
POST /api/v1/orders


- FastAPI（如需 HTTP API 服务）

- grpcio（gRPC 客户端，如需要）

### 3. 工具定义要求

每个工具必须：

1. 使用 `@tool` 装饰器
2. 有清晰的中文文档字符串
3. 参数有类型注解
4. 返回 JSON 字符串
5. 包含错误处理

**工具模板**：
```python
@tool
def tool_name(param: str, runtime: ToolRuntime = None) -> str:
    """工具描述。参数：param-参数说明"""
    try:
        # 1. 参数验证
        # 2. 调用 API
        # 3. 处理响应
        # 4. 返回 JSON
        return json.dumps(result, ensure_ascii=False)
    except Exception as e:
        raise LangChainException(f"友好的错误提示: {str(e)}")
```

### 4. API 客户端封装要求

**REST 客户端**：
- 封装为类，支持 GET/POST/PUT/DELETE
- 统一处理认证、超时、错误
- 使用环境变量配置

**gRPC 客户端**：
- 封装连接管理
- 统一错误处理
- 支持元数据认证

### 5. 枚举类型处理

如果 API 使用枚举值（如订单类型），必须：
1. 定义枚举映射表
2. 提供字符串到枚举的转换函数
3. 给出清晰的错误提示

```python
ORDER_TYPE_MAP = {
    "LO": "Limit",
    "MO": "Market",
}
```

### 6. 智能推断

对于股票代码等参数，实现智能推断：
- 支持 `TSLA` -> `TSLA.US`
- 支持 `700` -> `700.HK`
- 使用缓存提高性能

### 7. 错误处理

所有工具必须：
- 捕获所有异常
- 返回友好的错误提示
- 区分认证错误、网络错误、业务错误

### 8. 系统提示词

编写清晰的系统提示词，包括：
- 角色定义
- 核心能力列表
- 工作流程
- 交易规则
- 输出格式
- 约束条件

### 9. 用户手册

为每个工具提供：
- 功能说明
- 参数详解
- 用户提问示例（5个以上）
- 返回数据示例
- 注意事项

## 关键注意事项

### ✅ 必须做的
1. 所有工具调用真实 API，禁止 Mock
2. 参数必须有类型注解
3. 返回 JSON 字符串
4. 枚举类型必须转换
5. 每步都要测试验证

### ❌ 禁止做的
1. 禁止返回假数据
2. 禁止忽略错误
3. 禁止硬编码配置
4. 禁止跳过测试

### 💡 重要经验
1. 枚举类型转换为字符串再返回
2. 日期类型使用 `.isoformat()`
3. None 值提供默认值
4. 使用缓存减少重复请求
5. 敏感操作需要确认

```markdown

- 签名方式：HMAC-SHA256
```
